## 💬 範例對話

### 情境 1：建立簡單的資料上傳介面

**你可以這樣說：**

```
我想做一個網頁，讓使用者上傳 Excel 檔案，然後顯示資料摘要（幾行幾列、欄位名稱）
```

**進階要求（可選）：**

```
用 Streamlit 建立，包含檔案上傳區、資料預覽表格、基本統計資訊
```

```
加上資料篩選功能，讓使用者選擇要看哪些欄位
```

---

### 情境 2：建立互動式儀表板

**你可以這樣說：**

```
我想做一個銷售儀表板，使用者可以：
- 選擇日期範圍
- 選擇產品類別
- 看到圖表即時更新
```

**進階要求（可選）：**

```
左側放篩選條件（側邊欄），右側上方顯示關鍵數字卡片（總銷售額、訂單數），右側下方顯示圖表
```

```
加上 📅 日期範圍選擇器、🏷️ 產品類別多選框、💰 總銷售額顯示
```

---

### 情境 3：建立表單輸入介面

**你可以這樣說：**

```
我想做一個問卷表單，收集：
- 姓名（文字輸入）
- 部門（下拉選單）
- 滿意度（1-5 星評分）
- 意見（多行文字）
提交後儲存到 CSV
```

**進階要求（可選）：**

```
提交後顯示確認訊息，清空表單供下一位填寫
```

```
加上「已收到 N 筆回覆」的計數器
```

---

### 情境 4：建立多頁應用

**你可以這樣說：**

```
我想做一個完整的應用，包含多個頁面：
- 首頁（功能介紹）
- 資料上傳
- 數據分析
- 報告下載
```

**進階要求（可選）：**

```
使用者可以在側邊欄切換頁面，資料會在頁面間共享
```

## 💡 提示

### 介面設計的黃金原則

**1. 簡潔明瞭**

✅ 好的設計：
- 重要資訊放最上面
- 按鈕文字清楚（「上傳檔案」比「確定」好）
- 適當的留白

❌ 不好的設計：
- 把所有功能塞在一起
- 按鈕文字模糊
- 擁擠的版面

**2. 即時回饋**

✅ 好的體驗：
- 上傳檔案時顯示進度條
- 處理中顯示「載入中...」
- 完成時顯示成功訊息

**3. 錯誤處理**

✅ 友善的錯誤訊息：
```
檔案格式錯誤，請上傳 CSV 或 Excel 檔案
```

❌ 生硬的錯誤訊息：
```
ValueError: invalid file format
```

## 🎨 Streamlit 常用元件

### 輸入元件
| 元件 | 用途 |
|------|------|
| `text_input()` | 單行文字輸入 |
| `text_area()` | 多行文字輸入 |
| `number_input()` | 數字輸入 |
| `date_input()` | 日期選擇 |
| `file_uploader()` | 檔案上傳 |

### 選擇元件
| 元件 | 用途 |
|------|------|
| `selectbox()` | 下拉選單（單選） |
| `multiselect()` | 多選框 |
| `radio()` | 單選按鈕 |
| `checkbox()` | 勾選框 |
| `slider()` | 滑桿 |

### 顯示元件
| 元件 | 用途 |
|------|------|
| `write()` | 萬用顯示 |
| `dataframe()` | 互動式表格 |
| `metric()` | 指標卡片 |
| `code()` | 程式碼區塊 |

### 圖表元件
| 元件 | 用途 |
|------|------|
| `line_chart()` | 折線圖 |
| `bar_chart()` | 長條圖 |
| `plotly_chart()` | Plotly 互動圖表 |

### 版面配置
| 元件 | 用途 |
|------|------|
| `columns()` | 多欄布局 |
| `tabs()` | 分頁標籤 |
| `expander()` | 可展開區塊 |
| `sidebar` | 側邊欄 |

## 🌟 小技巧

### 如何描述介面需求

**範例 1：明確的版面配置**
```
我想要這樣的版面：
- 左側：篩選條件（日期、類別）
- 右側上方：關鍵數字卡片（總銷售額、訂單數）
- 右側下方：圖表（折線圖、圓餅圖）
```

**範例 2：具體的互動行為**
```
當使用者選擇不同的日期範圍，圖表要自動更新，並顯示該期間的統計數字
```

```
我想每 5 秒自動重新載入資料，不需要手動重新整理
```

```
使用者在不同頁面間切換時，之前上傳的資料要保留
```

### 快取機制

```
資料處理很慢，可以快取結果嗎？處理過的資料會被快取，下次載入會更快
```

## 🔐 安全性考量

分享給同事時注意：
1. 不要在程式碼中寫入密碼
2. 使用環境變數存放敏感資訊
3. 驗證使用者上傳的檔案格式
4. 限制上傳檔案大小
5. 考慮加上簡單的密碼保護
