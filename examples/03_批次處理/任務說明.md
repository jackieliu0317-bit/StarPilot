# ⚙️ 批次處理任務

## 🎯 任務目標

學習如何用自然語言讓 AI 幫你批次處理檔案：重新命名、格式轉換、內容修改。

---

## 📂 測試資料說明

資料夾中提供了測試檔案：

| 資料夾 | 內容 | 適合練習 |
|--------|------|----------|
| `待重新命名/` | 混亂命名的圖片（IMG_xxxx.jpg 等） | 批次重新命名 |
| `圖片資料/` | PNG、JPG、BMP 格式圖片 | 格式轉換 |

---

## 💬 範例對話

### 情境 1：批次重新命名

**你可以這樣說：**

```
我有一堆照片檔案，檔名很亂，想統一改成：旅遊_001.jpg、旅遊_002.jpg 這樣
照片在測試資料/待重新命名資料夾
```

**進階要求（可選）：**

```
按照檔案建立時間排序來編號
```

```
先預覽會改什麼，確認後再執行
```

💡 **重點技巧**：
- 指定具體的命名格式（前綴_流水號.副檔名）
- 要求預覽再執行，避免錯誤
- 指定排序方式（建立時間、修改時間、檔名）

---

### 情境 2：格式批次轉換

**你可以這樣說：**

```
我有很多 PNG 圖片想轉成 JPG，而且要壓縮檔案大小
```

**進階要求（可選）：**

```
壓縮品質設為 85%，儲存到「converted」資料夾，保留原始檔案不要刪除
```

```
把所有圖片轉換後壓縮到 500KB 以下
```

💡 **重點技巧**：
- 指定輸出格式和品質
- 指定輸出位置
- 決定是否保留原檔

---

### 情境 3：批次修改檔案內容

**你可以這樣說：**

```
我有 50 個 Word 文件，需要把所有「股份有限公司」改成「有限公司」
```

**進階要求（可選）：**

```
先幫我建立備份再開始修改
```

```
修改完成後給我一份報告，列出每個檔案改了幾處
```

💡 **重點技巧**：
- 批量修改前一定要備份
- 要求修改報告方便確認

---

### 情境 4：按規則分類檔案

**你可以這樣說：**

```
我有一堆檔案需要分類：
- 檔名有日期的（例如 2024-03-01_xxx）移到「有日期」資料夾
- 其他的移到「無日期」資料夾
```

**進階要求（可選）：**

```
把檔案按大小分類：
- 小於 1MB：移到「小檔案」
- 1MB-10MB：移到「中檔案」
- 大於 10MB：移到「大檔案」
```

```
按照副檔名分類到不同資料夾（PDF、Word、圖片、影片）
```

## 🎓 學到的技巧

1. **批次思維**：一次處理大量檔案，而不是一個一個手動處理
2. **規則定義**：清楚說明命名規則或處理邏輯
3. **安全優先**：重要操作前先備份或預覽
4. **自動化**：把重複性工作變成可重複使用的工具

## 💡 提示

### 批次重新命名的技巧

✅ **好的描述**：
```
改成：會議記錄_2024-01_001.txt
格式是：會議記錄_年月_流水號
```

❌ **不好的描述**：
```
改成有規則的檔名（什麼規則？）
```

### 格式轉換注意事項

轉換前建議確認：
1. 是否保留原檔案？
2. 轉換品質要求？（壓縮比例）
3. 輸出資料夾位置？
4. 檔名是否保持不變？

### 批次處理前的安全確認

```
可以先預覽會改什麼嗎？
```

## 🌟 小技巧

### 如何描述複雜的命名規則

**範例 1：包含多個元素**
```
檔名格式：部門_日期_流水號.副檔名
例如：業務部_2024-03-20_001.pdf
```

**範例 2：條件式命名**
```
如果檔案大於 5MB，加上「大檔」前綴
如果檔案小於 1MB，加上「小檔」前綴
```

### 批次處理的黃金法則

1. **先測試一個檔案**：確保邏輯正確
2. **一定要備份**：重要檔案處理前先備份
3. **分階段執行**：大量檔案分批處理
4. **記錄變更**：保存處理日誌方便追蹤

### 常見批次處理類型

| 類型 | 說明 | 範例 |
|------|------|------|
| 重新命名 | 統一命名格式 | IMG_001 → 產品照_001 |
| 格式轉換 | 圖片、文件、影片格式 | PNG → JPG |
| 內容修改 | 批次查找替換 | 舊公司名 → 新公司名 |
| 屬性修改 | 日期、權限、標籤 | 修改檔案日期 |
| 檔案分類 | 按規則移動或複製 | 按類型分資料夾 |
